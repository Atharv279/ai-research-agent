It proposes an innovative approach by representing these tasks as a graph of agent nodes, where each node represents a single round of execution with a fresh, clean context, and a supervisor node watches from outside and corrects drift.

Instead of having an agent that iterates over multiple rounds with shared context, graphkit uses a combination of executor nodes (which perform the work) and supervisor nodes (which watch from outside and correct drift). This approach allows for more accurate and reliable long-running coding tasks.
