The framework provides a role separation approach, where each agent performs a specific task, and the context lives only in memory.

**Key innovations**

What makes this project notable is its use of explicit queues to separate tasks among AI agents, promoting loose coupling and scalability. This differs from existing multi-agent frameworks that either rely on code-based approaches (e.g., LangGraph, AutoGen) or cloud platforms (e.g., CrewAI Studio, Vertex AI), which often come with vendor lock-in and high costs.

**Tech stack**

The primary language used is Python, with pyproject as the main framework. The Claude Code runtime is also utilized for full tool access and LLM gate functionality.

**Potential impact**

This project has the potential to affect the AI/ML ecosystem by providing an open-source, cost-effective alternative to existing multi-agent frameworks. By promoting role separation and explicit queues, aqm can facilitate the development of more scalable, modular, and maintainable AI systems. The use of YAML for pipeline declaration and shared access also makes it easier for users to create and reuse pipelines, potentially leading to increased collaboration and innovation in the field.
