Here is a concise summary of the GitHub repository "zero-drift" by obrenoalvim:

**Summary:**
The Zero Drift repository presents a Claude Code skill designed to prevent AI models from drifting (losing context) during long sessions. The skill consists of two rules:

1. **Named Response**: The AI opens every reply with the user's name, which serves as a hallucination detector, signaling when the model starts to drift.
2. **Living Task Document**: After each meaningful prompt, the AI records its actions, outputs, and exit codes in a `TASK.md` file, ensuring that context is preserved across sessions.

**Key Features:**

* Works with any markdown-reading AI (Cursor, Copilot, Codex)
* Maintains a clean context handoff between instances
* Uses JavaScript as the primary programming language

**Goals:**
Prevent AI models from drifting during long sessions and maintain a trustworthy record of progress and decisions made.
